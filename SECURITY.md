# Security Policy

## Public Safety & Privacy

The Interface Skills toolkit is an open-source project. To protect the privacy and security of users and contributors, please adhere to the following rules:

1. **No Sensitive Data:** Do not commit private screenshots, customer data, PII (Personally Identifiable Information), API keys, passwords, proprietary brand assets, or internal company documents.
2. **Sanitized Examples:** All examples, test scenarios, and spec packages should use fictional data or be thoroughly sanitized.
3. **Redacted Code:** If you are sharing code for a redline or inspection, ensure all proprietary logic or internal URLs are redacted or replaced with generic placeholders.

## Reporting a Vulnerability

If you discover sensitive data in the repository, its history, or a security vulnerability, please do not open a public issue. Instead, report it through the following channels:
