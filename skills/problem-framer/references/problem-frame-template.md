# Problem Frame

## 1. Raw Fog
The original, unedited idea or observation from the user.

## 2. Problem Under the Problem
The deeper root cause, desire, or technical tension identified during analysis.

## 3. Object Under Pressure
The specific part of the system, documentation, or workflow that is most affected or ambiguous.

## 4. Failure Mode
What will happen if this problem is not solved correctly? (The nightmare scenario).

## 5. Success Condition
What must be true for this to be considered solved? (The ideal outcome).

## 6. What Must Be True
List of assumptions or technical prerequisites that must hold for the success condition to be reachable.

## 7. Next Artifact
The recommended next sensemaking artifact (e.g., Unknowns Map, Repo Brief).
