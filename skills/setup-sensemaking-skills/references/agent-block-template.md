## Sensemaking Pipeline
This repository uses the `sensemaking-skills` pipeline to manage project uncertainty. Before implementing new features or fixing ambiguous bugs:

1. Use `repo-sensemaker` to diagnose the repository health and find the **Weakest Boundary**.
2. Follow the produced **Repository Sensemaking Brief**.
3. Use `workflow-orchestrator` to select a gated execution path.
4. Respect all **Approval Gates** defined in the orchestration plan.
