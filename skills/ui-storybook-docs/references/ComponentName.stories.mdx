# Component Storybook MDX Documentation: ProfileForm

This documentation covers the Storybook stories and setup for ProfileForm.

## 1. Components Documented

### Component 1: ProfileForm
- Description: The main form allowing user profile editing.

### Component 2: UserProfileBadge
- Description: Displays the user's avatar, display name, and email in the sidebar.

### Component 3: SettingsSidebar
- Description: Vertical sidebar containing links to all settings sections.

### Component 4: FormActions
- Description: Action buttons (Save, Cancel) with submit-in-progress indicator.

### Component 5: SettingsContent
- Description: Panel switching routes for settings views.

## 2. Ground Truth Context & Scope Mapping

This is the comprehensive source storybook documentation repository and component library setup for the Prosper Settings Page. It acts as the ground-truth reference for validating and syncing modern React component stories, properties, interaction testing, and inline markdown docs.

Our main goal is to generate, align, and sync storybook markdown files. We must model:
- ProfileForm: The complex profile detail and configuration form.
- UserProfileBadge: Sidebar widget for user profile avatar and metadata.
- SettingsSidebar: Main tabbed navigation panel for user options.
- FormActions: Standard sticky action bar matching submitting states.
- SettingsContent: Multi-pane content layout system.
