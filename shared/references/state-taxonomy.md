# State Taxonomy

This taxonomy standardizes the way we talk about state in both components and screens.

## Interactive States (Components)
- Default
- Hover
- Active / Pressed
- Focus / Focus-Visible
- Disabled

## Application States (Screens)
- Ideal State
- Empty State
- Loading State (Skeleton / Spinner)
- Partial Data
- Error State
- Offline State
- Permission Denied
